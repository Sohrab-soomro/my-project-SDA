# Trip Management System

A desktop-based Trip Management System developed in Java using the Java Swing framework for the Software Design and Analysis (SDA) course project.  
The system provides a complete environment for trip management, user authentication, request handling, and administrative operations through an interactive GUI.

---

## Project Overview

The application is designed to manage educational and institutional trips efficiently by supporting multiple user roles with different responsibilities and permissions.

The system follows important Software Design and Analysis concepts including:

- Object-Oriented Programming (OOP)
- SOLID Principles
- GRASP Principles
- Low Coupling and High Cohesion
- Modular Design
- Encapsulation and Abstraction
- GUI-Based Event Driven Programming

---

## User Roles

### 1. Student Portal
Students can:

- Login securely into the system
- View available trips
- Apply for trips
- Track trip requests
- Manage their personal information

---

### 2. Teacher Portal
Teachers act as trip supervisors and moderators.

Teachers can:

- View student trip requests
- Approve or reject applications
- Edit student trip requests
- Cancel requests if necessary
- Monitor student participation

---

### 3. Staff / Admin Portal
The Admin module manages the entire system.

Admin functionalities include:

- Announcing new trips
- Deleting trips
- Managing trip availability
- Activating or deactivating student accounts
- Approving student registrations
- Managing system records
- Controlling user access and permissions
